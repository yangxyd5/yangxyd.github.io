软硬链接的区别

类型 参数 支持路径 占用空间 跨磁盘性 特性
符号链接.symlinkd /d 相对/绝对 不占用 跨分区/跨网络 只是源目录的引用，源目录删除则不可用，链接文件删除不影响源文件，在操作系统层面解析，系统认为它是一个符号链接。
目录联接.junction /j 绝对 不占用 跨分区 同上
硬链接 /h 绝对 不占用 仅本分区 源文件的元数据级别引用，它们的数据块是同一个数据块，所以修改任意一个，另外一个也会发生变动，源文件删除不影响链接文件
快捷方式.lnk 右键 绝对 很小 跨分区 只是操作级别的解析，系统认为它只是一个文件

我的VBS脚本

我写了一个 VBS 脚本，方便链接的创建，主要实现了：智能判断源文件类型来实现创建软硬链接、拖放（支持多个文件）或者注册表右键的方式（传参问题！目前支持单个文件）调用，方便快捷

  • VBS脚本放在 C:\Windows\mklink.vbs
  • 添加右键菜单 mklink.reg
