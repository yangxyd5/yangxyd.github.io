Helm + storageclass + EFK

发表于 2019-12-31 | 分类于 Kubernetes

本文使用 K8S 集群上运行 EFK，使用 Ceph 集群作为 ElasticSearch 集群的持久存储。
用到知识有：Storage Class、PVC、Helm，另外，很多服务镜像需要翻墙。
helm install 阻塞过程会下载镜像可能会比较慢。
helm 里面有很多可以定制的项目，这里我就不定制了，反正我的资源也够用，懒得调了。
感觉没啥可以写的 😂
本文由于浏览量较多，所以我第二次更新些的更详细些，上一次是 5 月份。

Ceph 集群部署

本文主要针对 K8S 提供 StorageClass 而写的，主要为 K8S 提供两种存储能力：rbd（块存储）、cephfs（文件存储），这两种的区别就像一块硬盘与一个目录，在 K8S 上使用区别也只有读写模式上：ReadWriteOnce（rbd）、ReadWriteMany（cephfs）。
和 K8S 其他外部存储一样，每种存储需要一个 provisioner 插件，rbd 和 cephfs 各需要一个 provisioner。
这个笔记不打算写 K8S 是怎么部署的，我用的是 RKE 部署的。
另外网上有基于 K8S 的 ceph operator 好像已经进入稳定版本了，叫 rook-ceph ，有需要将 Ceph 托管在 K8S 集群上的小伙伴可以了解一下，由于太简单了，这里不说了。

K8S StorageClass

PVC 申请 PV 的时候，未必有符合条件的 PV，k8s 为我们准备了 StorageClass 可以在 PVC 申请 PV 的时候通过 StorageClass 动态生成 PV。

StorageClass 可以动态的到 CephFS 、NFS 等存储（或者云端存储）产生一个 PV，要求存储设备必须支持 RESTfull 风格的接口。
