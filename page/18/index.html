CSS介绍

工作模式

css 分为两个部分，选择谁、设置什么样式。

选择器决定了这个样式应用在那个目标上，大括号内每个 k, v 对都表示一个样式。

语法特性

  • 注释使用 / /

基本选择器

选择器决定了如何选择一个 HTML 元素， CSS 的基本选择器有：

  • 元素选择器(HTML元素名称：如：div、p)；
  • ID选择器（HTML元素的 ID 属性）；
  • 类选择器（HTML元素的 class 属性）。

元素选择器

标签选择器会选择所有该类型的标签，通常用于为标签设置默认的样式。

类选择器

为某一类元素，设置相同的类名称属性，然后 css 通过这个类名称来选择这些元素。

  • 类名称属性的特点

ID选择器

就是对某一个（注意就一个）元素，进行特别的样式设定的时候用。就是给元素加上 id 属性，然后在 css 选择时候用 # 号开头。

属性选择器

  • [attr]
    表示带有以 attr 命名的属性的元素。

  • [attr=value]
    表示带有以 attr 命名的属性，且属性值为”value”的元素。

  • [attr~=value]
    表示带有以 attr 命名的属性的元素，并且该属性是一个以空格作为分隔的值列表，其中[至少]一个值匹配”value”。

  • [attr|=value]

    表示带有以 attr 命名的属性的元素，属性值为“value”或是以“value-”为前缀（”-“为连字符，Unicode编码为U+002D）开头。典型的应用场景是用来来匹配语言简写代码（如zh-CN，zh-TW可以用zh作为value）。

  • [attr^=value]
    表示带有以 attr 命名的属性，且属性值是以”value”开头的元素。

  • [attr$=value]
    表示带有以 attr 命名的属性，且属性值是以”value”结尾的元素。

  • [attr*=value]
    表示带有以 attr 命名的属性，且属性值包含有”value”的元素。

  • [attr operator value i]
    表示带有以 attr 命名的属性, 且属性值匹配”value” [忽略属性值大小] 的元素。在带有属性值的属性选型选择器表达式的右方括号, 前添加用空格间隔开的字母i（或I）可以忽略属性值的大小写（ASCII字符范围内的字母）

高级选择器

按照编程惯例， CSS 一般使用元素的 class 属性来作为选择依据，而元素的 id 属性按照惯例是属于 javascript 选择器来使用的，CSS 除了上述的基本选择器，还有一些高级选择器，它们由基本选择器的组合，来提供更灵活的选择方式。

后代选择器

在选择一个具有 class 属性的元素的后代元素时候，可以空格跟上后代元素名称，就可以选择这些后代元素了。

注意：选择的目标是后代的元素，无论是子标签，还是孙子标签，或者更深，都能被选择。

  • 可以多重描述祖先的存在顺序，但是必须存在真实的顺序。

交集选择器

一般交集选择器语法为：标签名.类名，注意中间没有空格，这表示被选择的目标为：既是某种标签又包含某个类才会被选择。

并集选择器

使用逗号隔开的两部分表示并集，表示这两个部分内容都会被选择。

在 Google Chrome 浏览器中审查元素会发现：Inherited from div，表示元素从 div 继承了样式。

层叠性

一个元素可以同时从多个选择器里得到样式，样式是一层一层的抹上去的，当多个选择器都规定了元素的一个样式，CSS 的层叠性决定了如何处理不同选择器样式定义的冲突。

在 Google Chrome 浏览器中审查元素会发现：有些样式被横杠划去，这些就是由于 层叠性冲突 而被覆盖掉的样式。

选择器的优先级

CSS 层叠性会在不同选择器都 定义了相同的样式 时候，通过选择器的优先级来决定最终那个选择器的样式会胜出，选择器的优先级按照 由高到低 次序排列分别为：ID选择器 > 类选择器 > 元素选择器。

  • 多个选择器都规定了一个元素的某个样式时候，可以按照优先级来决定哪个生效

高级选择器权重

高级选择器由多个基本选择器组合而成，通过计算它包含的基本选择器的数量，就知道一个高级选择器的权重是多少了；

计算公式为：ID选择器的数量, 类选择器的数量, 元素选择器的数量 = 权重，例如：#box1 .div1 p，权重为：1,1,1；

对比每个高级选择器的权重，需要注意：低权重数量无论有多少个，都不能与高权重相比，最后有最多高权重选择器数量的生效。

原子类权重提升

很多时候我们会定义一些原子类（通用性的，原子的），这些原子类在使用的时候可能因为权重过低而无法生效，所以就需要将原子类的样式权重最大，以实现将原子类定义的样式层叠给其他类，这时候可以在样式后面使用 !important 关键字，将这个属性权重设置为无穷大。

原子类是一种编写 CSS 的思想，即：抽出高度复用的样式模块，独立成一个原子类，为对应的元素添加。

原子类在编程中，有人主张使用，因为高度抽象，可以精简胆码，而有人不主张使用，因为增加了解读它的时间，不利于维护。

background-position 图片定位

background-position 属性设置背景图像的起始位置，利用定位功能可以实现在一张大的图片上裁剪一个小的部分，这个技术叫精灵技术（CSS雪碧）。可以降低 HTTP 请求数量，从而降低服务器压力。

值 描述
位置1 位置2 可用：top bottom leftl right lcenter 组合，如果仅设置了一个关键词，那么第二个值将是 “center”。
默认值：0% 0%。
x% y% 第一个值是水平位置，第二个值是垂直位置。左上角是 0% 0%。右下角是 100% 100%。
如果您仅规定了一个值，另一个值将是 50%。
xpos ypos 左上角是 0% 0%。右下角是 100% 100%。第一个值是水平位置，第二个值是垂直位置。左上角是 0 0。
单位是像素 (0px 0px) 或任何其他的 CSS 单位。
如果您仅规定了一个值，另一个值将是50%。您可以混合使用 % 和 position 值。
  • 精灵定位

一般网站的小图标都是用的一整张图，然后在一个固定大小的盒子内，用 background-position 定位到这个图片的一部分。

首先用 fireworks 打开图片，使用裁剪工具定位为想要显示的图片片段，然后在左下角可以看到这个区域处于图片的位置，一般使用 background-position -Npx -Npx; 就可以定位到这个图片了。

文档流指的是元素排版布局过程中，元素会默认自动从左往右，从上往下的流式排列方式。并最终窗体自上而下分成一行行，并在每行中从左至右的顺序排放元素。

标准文档流的性质

  • 空白折叠
  • 自动换行
  • 高矮不齐，底边对齐

块级元素和行内级元素

标准文档流把 HTML 标签分为两大类：块级元素、行内级元素

性质 元素
块级元素（block level） div、h系列、p、ul、li、dl、dt、dd
行内级元素（inline level） span、a、b、u、i、em、strong
  • 块级元素（block level）性质
  • 行内级元素（inline level）性质
  • 元素性质转换

块级元素和行内级元素可以通过CSS display 属性来自由转换，转换后的元素拥有其所有性质。

一般块级转为行内没有什么用，而行内转块级就非常有用了，下面利用 a 标签制作一个更大的按钮。

float 浮动

浮动的特性

脱离标准流有三个方法：浮动、绝对定位、固定定位，浮动的元素 脱离标准流 ，所以就没有标准流的行块（inline、block）之分了。

行块二象性

在标准流中一个块级元素，不设置宽度则自动撑满父亲的宽度，但是在非标准流中一个块级元素，如果不设置宽度就会像行内元素一样收缩为内部文字的宽度。如果设置了宽度就会像块级元素占用设置的宽度，这个特性叫：行块二象性。

元素依序贴靠

如果之前的兄弟元素后面已经没有容纳自己的空间，那么将依次向上一个兄弟寻找足够的空间。

元素 margin 不塌陷

margin 塌陷现象，是标准流里的特性，所以浮动后的元素没有 margin 塌陷现象了。

不占用标准流

浮动的元素不占用标准流的位置，所以处于标准流元素的上方。

标准流字围特性

当浮动的元素处于标准流元素上方，且标准流元素内容为文字，则文字会围绕在浮动元素的周围。

不撑起标准流的高度

当父元素是没有高度的标准流元素，而子元素是有高度浮动元素，这时子元素的高度不能够撑起让父元素产生高度。

浮动的元素不能 margin

overflow 内容溢出边框

当元素的内容超出了元素的边框，则 overflow 属性就规定了如何处理这些超出边框的内容。

值 描述
visible 默认值。内容不会被修剪，会呈现在元素框之外。
hidden 内容会被修剪，并且其余内容是不可见的。
scroll 内容会被修剪，但是浏览器会显示滚动条以便查看其余的内容。
auto 如果内容被修剪，则浏览器会显示滚动条以便查看其余的内容。
inherit 规定应该从父元素继承 overflow 属性的值。
  • 超出元素边框的部分会被隐藏掉，但是超出 padding 的不会处理的。

让浮动元素撑起父亲高度

overflow 属性除了可以处理超出边框的内容，还能让父元素检查浮动的子元素的高度，然后避免超出，这实现了浮动元素撑起父元素的效果。

  • 利用 overflow 属性使浮动的元素撑起父元素高度

清除不同盒子内浮动

当两个标准流父元素有多个浮动元素，且父元素没有设置高度的时候，那么第二个父元素内的浮动元素就有可能跑到第一个父元素盒子的内部，造成布局混乱。此时可以在父元素样式中使用 overflow 来使得父元素有高度，从而不会让外部浮动元素进入。

clear 清除两侧浮动

clear 属性应该加给浮动元素的父元素，以让父元素清理外部不属于自己的浮动元素。

  • clear 可用值
值 描述
left 在左侧不允许浮动元素。
right 在右侧不允许浮动元素。
both 在左右两侧均不允许浮动元素。
none 默认值。允许浮动元素出现在两侧。
inherit 规定应该从父元素继承 clear 属性的值。

清除浮动让多个浮动有序

当两个标准流父元素有多个浮动元素，且父元素没有设置高度的时候，那么第二个父元素内的浮动元素就有可能跑到第一个父元素盒子的内部，造成布局混乱。此时可以在父元素样式中使用 clear: both; 来清除不属于自己的浮动元素。

  • 需要注意

标准流父元素没有高度，那么 clear 可能造成标准流父元素的之间的 margin-top margin-bottom 被压缩，除非 margin 高于浮动元素。

直接对父盒子使用 clear 会造成父元素的之间的 margin-top margin-bottom 被压缩，所以不常用。

隔墙法清除之间浮动影响

当两个标准流父元素有多个浮动元素，且父元素没有设置高度的时候，那么第二个父元素内的浮动元素就有可能跑到第一个父元素盒子的内部，造成布局混乱。此时可以在两个标准流元素之间增加一个块级元素，来隔离两个标准流元素内部浮动元素的影响。

  • 隔墙法还有一个好处就是对墙设置的 margin 是有效的

外墙法一般用于两个特别大的盒子之间，还可以充当空白

内墙法清除之间浮动影响

将清除浮动的墙设置在每个标准流元素内部，能清除不属于标准流元素的浮动元素进入标准流元素，同时 margin 还可以生效。

内墙法有代码侵入，一般用的不多

清除浮动使用场合总结

  1. 工作中，我们总喜欢给内部有浮动的父盒子加上 overflow: hidden; 。
  2. 我们总喜欢在两个大的父盒子之间放一个当作隔墙的盒子，一般为外墙。
  3. 工作中，当盒子内部有浮动元素，我们一般不为盒子写高度，而是写 overflow: hidden; ，让内部浮动元素撑起父盒子。

position 定位

CSS 三大布局知识：盒模型、浮动、定位

relative 相对定位

相对定位就是相对这个盒子原来的位置进行定位，相对定位不脱离标准流，而原来的位置不会被别的元素占用，且作用于原来位置的 margin 仍然会对其他元素产生效果。

相对定位导航条上边框

absolute 绝对定位

绝对定位就是相对整个页面的位置进行定位，脱离标准流。

使用 top，bottom，left，right 来进行定位，参考的是页面的左上角，无视参考点的 padding。

  • top 相对整个页面的顶部
  • bottom 相对第一屏的底部
  • 绝对定位理解图

如果用 top 定位就是页面的左（右）上角。

如果用 bottom 定位 就是第一屏的左（右）下角。

在父盒子内绝对定位

如果给父辈盒子设置相对（绝对、固定）定位属性，那么子盒子的绝对定位属性将是相对于离得最近的有定位属性的父辈盒子而言的。

一般父元素是相对定位（不脱离标准流），子元素采用绝对定位。也就是俗称的：子绝父相。

绝对定位元素居中

脱离标准流的元素，不能使用 margin 0 auto; 来进行居中，所以绝对定位的元素居中不能使用 margin 0 auto; 来居中。

  • 水平居中

下面的示例：left 50%，父元素的左边50% 地方开始，margin-left -300% 指的是外左边距不向外，而向内 -300。

  • 垂直居中

下面的示例：top 50%，父元素上边 50% 的地方开始，margin-top -300% 指的是上外边距不向外，而向内 -100。

img 大图片居中

图片的分辨率为：1920 × 566 的，css 的 img 是行内级元素，默认从左到右显示，当屏幕小的时候就会造成横向溢出，产生横向滚动条。

fixed 固定定位

固定定位就是相对屏幕窗口的位置进行定位，不会随窗口个滚动而消失，脱离标准流。

  • 使用固定定位，将标题栏固定到屏幕的一个地方，不随滚动条滚动。

高级效果需要 js 来实现，例如导航条处于网页中间，当用户滚动屏幕直到这个条处于屏幕顶部的时候，这个导航条被 fixed 固定。

脱离标准流总结

脱离标准流的方法有三种：

  • 浮动

  • 绝对定位

  • 固定定位

这三个只能有一个作用在元素上，不能同时存在同时浮动、同时绝对定位的情况；

我们给一个父元素 overflow:hidden，这个时候父亲就能认识浮动的子元素的高度了，就能被儿子撑高了；

但是，绝对定位、固定定位的儿子，父亲永远不可能被他们撑高。

Z-index 覆盖顺序

负责设置压盖顺序，谁压盖谁的问题，默认的压盖顺序是：

  • 定位的能够压住没有定位的
  • 如果都定位了 HTML 顺序后面的能够压盖住前面的

zindex 值的设置

  • 且只有定位了的元素才能写 z-index 值；

  • 标准流的元素不能写 z-index 值。

  • z-index 值没有单位，数字大的能够压住数字小的；

  • 如果父亲的 z-index 值小的话，那么子元素无论多大都无效；

盒子透明

opacity 是调节背景颜色的不透明度。0 -> 1 即：透明 -> 不透明。

图片透明

  • jpg、jpeg

jpg/jpeg 压缩格式，为了保存尺寸小，所以有压缩算法，颜色是失真的，网页中的照片、新闻图片、banner、焦点图、都要用 jpg 格式的、因为这样 保存图片格式更小。

  • png

不可压缩、颜色不失真、是 fireworks 软件的默认保存格式、可以保存图层，支持透明和半透明（无关图层）

在传到服务器上面的时候，所有的 png 图片都应该去掉图层，去掉的方法就是 fireworks 的导出功能。

一般 jpg 图片比 png 图片的体积小，但是 png 更善于处理矢量的、形状规则内容的图形，所以，如果有很多几何图形的图片可以保存为 png ，它的体积会比 jpg 更小些，且质量更高。

  • gif

gif 它只能支持固定数量的颜色，可以是 256、128、64 种，所以它根本不是压缩不压缩的问题，它根本无法表达出更多的颜色，所以是严重的失真。

但是，它支持动画，支持透明（不支持半透明）。

尺寸比较小，因为颜色数量少。

fireworks 输出透明的 gif ：在优化里选择：GIF动画、色板选为透明、选择精确

  • bmp

不压缩的，不失真的，没有透明半透明，没有图层。
