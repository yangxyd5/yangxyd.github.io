命名空间

由于在 Python 中一切皆为对象，想要好好理解 Module 和 Package，一定要先理解 Namespace 的概念。 所谓 Namespace ，是指标示符的可见范围。对于 Python 而言，常见的 Namespace 主要有以下几种：

  1. Build-in Namespace (内建命名空间）
  2. Global Namespace (全局命名空间）
  3. Local Namespace (局部命名空间）

有了命名空间的概念，可以有效的解决函数或者是变量重名的问题。不同的命名空间中允许出现相同的函数名或者 是变量名。它们彼此之间不会相互影响，例如在Namespace A和B中同时有一个名为var的变量，对A.var赋值并不 会改变B.var的值。

模块

一个包含Python代码并且以.py结尾的文本文件就是一个模块，其文件名部分就是模块名，其中定义的所有变量或者函数都属于这个Module，这个Module 对于所有函数而言就相当于一个全局的命名空间，而其中的每个函数又都有自己局部的命名空间。

包

包是一个目录，并且必须有一个__init__.py文件，它是包的注册文件，文件内容可以为空，一个包内可以有多个 模块 或者 子包 组成，这样一个有层次的目录结构形成了Python的包，可以将包视作模块。

导入包会执行包下的__init__.py注册文件。

导入包下的子模块，一定会先导入__init__.py注册文件，然后再导入子模块的代码。

导入包下的子包，则不会导入父包，只会导入子包的__init__.py。

相关属性

搜索路径

即：导入模块时在哪些路径下搜索模块，搜索路径是编译 Python 时候指定的，它以一个列表的形式保存在sys.path变量里，其中每个元素表示一个路径，对这个列表 append ，可以增加搜索路径。

模块属性

属性 含义
__file__ 被引用模块的路径
__cached__ 编译后字节码文件路径
__spec__ 模块的规范
__name__ 是模块内置的一个属性，一般如果模块是被引用的时候, 它的值是模块名, 如果这个python文件被直接运行, 那么它的值是'__main__'
__package__ 如果文件是个包它的值和__name__是相同的，如果是子模块，则和父模块一致
__all__ 在包的注册文件内，它的值是个模块名组成的列表，该属性表示当使用import * from xxx时候，哪些模块会被导入进来
__doc__ 模块的注释文档

隐藏变量

如果不想让某个属性被from module import *导入，那么你可以给不想导入的属性标识符前面加_，但这不能组织import foo.__bar这种形式的导入。

打包分发

  • setuptools模块

setuptools 可以说是 Python distutils 的增强版，它能帮助我们更好的创建和分发 Python 的包，尤其是具有复杂依赖关系的包。对于开发者来说，能够更好的组织自己项目的分发和发布；对于用户来说，不需要安装 setuptools 也可以使用由它创建的包，只需要一个启动模块即可。

实现这样的的包管理机制主要由两部分构成：

  • 一个存储在 Python 官方网站的集中式仓库，名叫 Python Package Index（PyPI）
  • 另外就是基于 distutils 开发的 setuptools 包管理系统

hashlib加密解密

发表于 2017-10-08 | 分类于 Python

摘要算法又称哈希/散列算法。它通过一个函数，把任意长度的数据转换为一个长度固定的数据串（通常用16进制的字符串表示）。算法不可逆。

散列/哈希算法有：

MD5

MD5（Message-Digest Algorithm 5，消息摘要算法版本5）是最常见的摘要算法，它由MD2、MD3、MD4发展而来，速度很快。

MD2、MD4、MD5 都产生16字节（128位）的校验值，一般用32位十六进制数表示。MD2的算法较慢但相对安全，MD4速度很快，但安全性下降，MD5比MD4更安全、速度更快。

SHA1

SHA（Secure Hash Algorithm）是由美国专门制定密码算法的标准机构—— 美国国家标准技术研究院（NIST）制定的。

SHA系列算法的摘要长度分别为：SHA为20字节（160位，通常用一个40位的16进制字符串表示）、SHA256为32字节（256位）、 SHA384为48字节（384位）、SHA512为64字节（512位），由于它产生的数据摘要的长度更长，因此更难以发生碰撞，因此也更为安全，它是未来数据摘要算法的发展方向。由于SHA系列算法的数据摘要长度较长，因此其运算速度与MD5相比，也相对较慢。

SHA1的应用较为广泛，主要应用于CA和数字证书中，另外在互联网中流行的BT软件中，也是使用SHA1来进行文件校验的。
