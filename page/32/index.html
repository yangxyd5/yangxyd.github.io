SocketServer

SocketServer 模块简化了编写网络服务器的任务，它有一个超类 BaseServer 负责定义API，又通过继承重写方式定义出了四个子类。

子类 作用
BaseServer 定义了核心服务器功能的API，但不具体实现
TCPServer TCP流式套接字
UDPServer UDP数据报套接字
UnixStreamServer Unix形式的TCP套接字
UnixDataramServer Unix形式的UDP套接字

这四个字类之间的继承关系是：

这四个字类，它们以同步方式处理用户请求，也就是说处理完一个用户请求才会处理下一个用户的请求，如果每个请求都需要很长时间才能完成，这就非常不合理了，解决方案是创建一个单独的进程或线程来处理每个请求，所以 socketserver 模块还有两个混入类 ForkingMixIn 和 ThreadingMixIn 用于支持异步方式。

  • 经过混入的方式又出现了四个字类
字类 作用
ForkingTCPServer 多进程的TCP流式套接字
ForkingUDPServer 多进程的UDP数据报套接字
ThreadingTCPServer 多线程的TCP流式套接字
ThreadingUDPServer 多线程的UDP数据报套接字

Server Objects

服务器对象

class socketserver.BaseServer(server_address, RequestHandlerClass) 这是模块中的所有服务器对象的超类。它定义了接口，如下所述，但是大多数的方法不实现，在子类中进行细化。

方法\ 属性 含义
fileno() 返回套接字文件描述符
handle_request() 该方法用于处理单一的 request，按顺序调用get_request(), verify_request() process_request()
serve_forever(*poll_interval=0.5*) 处理请求，直到一个明确的 shutdown() 请求。每 poll_interval 秒轮询一次 shutdown 。忽略 self.timeout。如果你需要做周期性的任务，建议放置在其他线程。
service_actions() 它在 serve_forever() 循环中调用，此方法可由子类别或 mixin 类重写, 以执行特定于给定服务 (如清除操作) 的操作。
shutdown() 告诉 serve_forever() 循环停止并等待其停止。
server_close() 清理服务器，可能被重写
address_family 地址家族，比如 socket.AF_INET和 socket.AF_UNIX。
RequestHandlerClass 需要用户重写的请求处理类，这个类为每个请求创建实例。
server_address 服务器侦听的地址，格式根据协议家族地址的各不相同，请参阅socket模块的文档。
socket 服务器上侦听传入的请求socket对象的服务器。

  • 服务器支持的类变量
    | 方法|属性 | 含义 |
    | ——————— | ———————————————————— |
    | allow_reuse_address | 服务器是否允许地址的重用。默认为false ，并且可在子类中更改。 |
    | request_queue_size | 请求队列的大小。如果单个请求需要很长的时间来处理，服务器忙时请求被放置到队列中，最多可以放request_queue_size个。一旦队列已满，来自客户端的请求将得到 “Connection denied”错误。默认值通常为5 ，但可以被子类覆盖。 |
    | socket_type | 服务器使用的套接字类型，socket.SOCK_STREAM和socket.SOCK_DGRAM等。 |
    | timeout | 超时时间，以秒为单位，或 None表示没有超时。如果handle_request() 在 timeout 内没有收到请求，将调用handle_timeout()。 |

  • 可被字类重载的方法
    | 方法|属性 | 含义 |
    | —————————————— | ———————————————————— |
    | finish_request() | 实际处理 RequestHandlerClass发起的请求并调用其handle()方法。 常用。 |
    | get_request() | 接受socket请求，并返回二元组包含要用于与客户端通信的新socket对象，以及客户端的地址。 |

    | handle_timeout() | 超时处理。默认对于 forking 服务器是收集退出的子进程状态，threading 服务器则什么都不做。 |
    | process_request(request, client_address) | 调用 finish_request() 创建 RequestHandlerClass 的实例。如果需要，此功能可以创建新的进程或线程来处理请求, ForkingMixIn 和 ThreadingMixIn 类做到这点。常用。 |

Request Handler Objects

请求处理器，处理器接收数据并决定如何操作。它负责在socket层之上实现协议（i.e., HTTP, XML-RPC, or AMQP)，读取数据，处理并写反应。可以重载的方法如下：

方法/属性 作用
setup() 当连接套接字建立，服务器为本次请求做的准备工作. 默认什么都不做，StreamRequestHandler中会创建文件类似的对象以读写socket。
handle() 处理本次连入的请求，解析传入的请求，处理数据，并发送响应。默认什么都不做。常用变量：self.request，self.client_address，self.server。
finish() 连入的请求断开时候，执行的方法，默认什么都不做，如果 setup() 产生异常，不会执行finish()。
.request 本次连入请求的socket对象，支持：.request.recv、.request.send 等
.client_address 本次连入的请求的客户端地址
.server sockerserver对象

通常只需要重载handle，重写setup()和finish() ，并提供self.rfile和self.wfile属性，self.rfile和self.wfile可以读取或写入，以获得请求数据或将数据返回到客户端。

self.request的类型与服务类型相关，对于TCP服务self.request 是 socket 对象；对于UDP服务，self.request是字符串和socket，可以在子类StreamRequestHandler或DatagramRequestHandler中重载，

SocketServer 内部使用了 selectors 方式实现异步的通信。

在一次通信中，每一个请求会对应一个BaseRequestHandler实例，由它负责处理用户的请求。

创建SocketServer

  1. 创建一个类，它继承自 socketserver.BaseRequestHandler 并且在类中重写以覆盖父类的 handle 方法。
  2. 实例化一个SocketServer 如：sockserver.ThreadingTCPServer，并且传递两个参数：(IP,PORT) 和 请求处理类 给这个实例。
  3. 调用这个 SocketServer 的 serve_forever(*poll_interval=0.5*) 方法，来启动这个 SocketServer 。
  4. 调用SocketServer.close()

语法：get(self, block=True, timeout=None)

block：阻塞标志，表示是否阻塞。

timeout：阻塞时间，秒。

非阻塞读出.get_nowait
