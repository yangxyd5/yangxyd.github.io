位运算

位运算，由于当前计算机是基于电信号存储/处理信息，而电信号只有高电平和低电平，分别被计算机标志为1，0. 对于十进制的数字，比如1，在早期的8位计算机中，存储为诸如 00000001 表示，这样就可以做一些有用的计算，比如通过移动位，比如整体左移或右移，就可以神奇的改变数值。

Option Explanation
& 按位与运算符：参与运算的两个值,如果两个相应位都为1,则该位的结果为1,否则为0。
\ 按位或运算符：只要对应的二个二进位有一个为1时，结果位就为1。
^ 按位异或运算符：当两对应的二进位相异时，结果为1。
~ 按位取反运算符：对数据的每个二进制位取反,即把1变为0,把0变为1 。~x 类似于 -x-1
<< 左移动运算符：运算数的各二进位全部左移若干位，由 << 右边的数字指定了移动的位数，高位丢弃，低位补0。
>> 右移动运算符：把”>>”左边的运算数的各二进位全部右移若干位，>> 右边的数字指定了移动的位数

字符串

单个英文字母被称为是一个字符，实际上计算机的字符是指 ASCII 码，但 Python 没有 C 语言的 char 的数据类型的概念，所以单个字符实际上也是字符串。由于汉字一般被编码为两个字符（PythonUnicode 编码为 3 个），所以汉字不能说是一个字符，而是一个 “字”。

Python使用 单引号，双引号，单三引号，双三引号 定义的一组字母，数字，符号（非特殊系统符号）的集合，称之为字符串，在字符串前加上r/R前缀，表示该字符串不做特殊处理。

  • str表示字符串类型
  • 引号的区别使用
  • 不可变性质

字符串 (str) 存储 Unicode 文本，是不可变序列类型，支持序列的大部分操作，索引，分片，乘法，加法，判断成员资格，求长度，最小值，最大值，同样适用于字符串，唯一需要注意的是字符串是不可变的对象。

列表

列表英文为list，是Python的一种基本的序列数据结构，它是线性数据结构，列表是可变对象，可以在任意位置插入数据，列表中的每个元素被分配一个序号，这个序号代表元素所处的位置，也称之为索引，索引从0开始计数，也支持-1这样的负数索引来倒序引用。

列表的定义格式为：list_var_name = [元素1，元素2，元素3]

list_var_name为列表的名字，列表中的元素使用逗号分隔，外侧使用方括号括起来，元素的类型可以是字符串、数字、也可以是：其他的列表、元组、字典等类型的数据结构。

Python简介

Python 的设计具有很强的可读性，相比其他语言经常使用英文关键字，其他语言的一些标点符号，它具有比其他语言更有特色语法结构。

  • Python 是一种解释型语言：这意味着开发过程中没有了编译这个环节。类似于PHP和Perl语言。
  • Python 是交互式语言：这意味着，您可以在一个Python提示符，直接互动执行写你的程序。
  • Python 是面向对象语言: 这意味着Python支持面向对象的风格或代码封装在对象的编程技术。
  • Python 是动态、强类型语言：这意味着不用给变量指定数据类型，它会在第一次赋值给变量时在内部将数据类型记录下来。

垃圾回收

Python使用引用计数方式来进行垃圾回收的，当一个对象引用计数小于1的时候将被使用标记清除的方法从内存中清除，标记清除就是将对象标记为空白，这种算法的缺点是会产生内存碎片。

基础语法

  1. 转义序列：\,\t,\r,\n,\’,\”，具体见：man ascii，前缀r/R会让转义序列失效。
  2. 缩进：Python使用缩进的层级来表示代码层次关系。
  3. 代码续行：在行尾使用\，来表示一行未完成，下一行将继续，如果使用括号，认为括号内是一个整体。
  4. python 的变量名自定义函数名，只能包括字母数字下划线，且数字不能开头。
  5. 不能使用 python 内部的关键字，例如：def，class，等。

数据类型

Python的对象从概念上大致分为四类：

数值对象（Math）：

容纳其他对象的集合对象（Contaniner）

表示程序结构的对象（Composition）

Python内部对象（Internal）

数据结构

  1. 列表：Python关键字标记为 list，用来表示“顺序的数据”
  2. 元祖：Python关键字标记为 tuple，用来表示“不可修改的顺序数据”
  3. 字典：Python关键字标记为 dict，用来表示“有别名的数据”
  4. 集合：Python关键字标记为 set，用来表示“没有顺序的数据”

非内置数据结构

  1. 堆：通常是由完全二叉树实现，可以看作一棵树的数组对象，堆是应用程序在运行的时候请求操作系统分配给自己内存，一般是申请/给予的过程，需要注意的是堆是在程序运行时，而不是在程序编译时。
  2. 栈：正好与队列相反，它实现了后进先出(LOFO)，对于堆栈的所有数据操作，都是在栈的顶部完成的，只可以查看栈顶部的数据，只能够向顶部压入数据，也只能在顶部弹出数据，这个结构主要用于递归。
  3. 队列：用来表示“先进先出”的数据(FIFO)，形象的比喻就像“毛毛虫”，它是先吃的东西，先消化拉出去。在开发中常用的消息处理，就是先来的消息，先“消化”，最后来的最后“消化”。
  4. 链表：链表是在非连续的内存单元中保存数据，并且通过指针将各内存单元相链接在一起，链表不需要事先申请固定大小的内存空间，对于大数据检索等比较常用。
  5. 二叉树：表示父子关系的数据，就更加抽象了，暂不了解。
