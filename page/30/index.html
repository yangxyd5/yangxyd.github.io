err类

异常 描述
Warning 当有严重警告时触发，例如插入数据是被截断等等。必须是 StandardError 的子类。
Error 警告以外所有其他错误类。必须是 StandardError 的子类。
InterfaceError 当有数据库接口模块本身的错误（而不是数据库的错误）发生时触发。 必须是Error的子类。
DatabaseError 和数据库有关的错误发生时触发。 必须是Error的子类。
DataError 当有数据处理时的错误发生时触发，例如：除零错误，数据超范围等等。 必须是DatabaseError的子类。
OperationalError 指非用户控制的，而是操作数据库时发生的错误。例如：连接意外断开、 数据库名未找到、事务处理失败、内存分配错误等等操作数据库是发生的错误。 必须是DatabaseError的子类。
IntegrityError 完整性相关的错误，例如外键检查失败等。必须是DatabaseError子类。
InternalError 数据库的内部错误，例如游标（cursor）失效了、事务同步失败等等。 必须是DatabaseError子类。
ProgrammingError 程序错误，例如数据表（table）没找到或已存在、SQL语句语法错误、 参数数量错误等等。必须是DatabaseError的子类。
NotSupportedError 不支持错误，指使用了数据库不支持的函数或API等。例如在连接对象上 使用.rollback()函数，然而数据库并不支持事务或者事务已关闭。 必须是DatabaseError的子类。

Cursor Objects

方法 作用
.callproc( procname [, parameters ] ) 调用存储过程并且传递参数，结果集可以通过 .fetch*() 得到
.execute(operation [, parameters]) 执行数据库命令，.execute('operation', (parameters)) 使用类似C风格的字符串格式化方法，可以防止SQL注入攻击
.executemany( operation, seq_of_parameters ) 批量操作，.execute('operation', (seq_of_parameters)) 这里的参数可以是可迭代对象组成的序列，这会调用 operation 对序列中的每个元素执行操作
.fetchone() 获取查询结果集的下一行，返回单个序列，或在没有更多数据可用时返回None
.fetchmany([ size = cursor.arraysize ]) 获取查询结果的下一组行，返回一系列序列（例如元组列表）。没有更多行可用时返回空序列。
.fetchall() 获取查询结果的所有（剩余）行，并将它们作为一系列序列（例如元组列表）返回。请注意，游标的arraysize属性可能会影响此操作的性能。
.close() 关闭游标。也可以直接关闭连接，从而关闭游标
.nextset() 此方法将使光标跳到下一个可用集合，丢弃当前集合中的任何剩余行
.setinputsizes(sizes) 这可以在调用 .execute*() 之前使用，以预定义操作参数的内存区域。
.setoutputsize(size [，column ]) 为大列提取设置列缓冲区大小（例如，LONG s，BLOB s等），该列被指定为结果序列的索引。不指定列将为游标中的所有大型列设置默认大小
.scroll(value [, mode='relative' ]) 根据 mode 移动游标到新位置，默认 mode ='relative' 表示相对当前的偏移量，当为 mode ='absolute' 时移动到绝对位置
next() 返回下一行数据
.mogrify(query,args=None) 参数化查询，防止SQL注入攻击，通过调用 execute() 方法返回发送到数据库的确切字符串。
属性 作用
.description 游标结果的字段描述信息，包含了：字段名，字段类型代码，显示尺寸，内部大小，精确度，规模，是否允许空
.rownumber 显示游标所在行
.rowcount 这是一个只读属性，并返回执行 execute() 方法后影响的行数
.arraysize 此读取/写入属性指定一次使用 .fetchmany() 提取的行数。它默认为1
.connection 此只读属性返回创建光标的Connection对象的引用。
.__iter__ 将游标转换为可迭代对象
.lastrowid 获取新创建数据的自增ID
