try语法

try过程

  • try 会开启一个上下文，然后执行 try 与 except 之间的语句，语句将决定控制流能否正常通过try
  • 如果 try 与 except 之间的语句发生异常，则立即从异常处跳出到 try 上下文，try 语句捕获异常后，将异常交给第一个匹配的 except 子句，执行except子句的代码，完毕后返回 try ，然后控制流正常通过 try 语句
  • 如果 try 与 except 之间的语句发生异常，则立即从异常处跳出到 try 上下文，try 语句捕获异常后，没有找到匹配的 except 子句，则 try 会将异常向上层抛出，并且不再向后执行同级代码。
  • 如果 try 与 except 之间的语句没有发生异常，则 try 语句会执行 else 子句的代码，然后控制流正常通过 try 语句
  • 无论程序控制流是否正常通过try语句，都会在最后执行finally 子句

柯里化

在计算机科学中，柯里化（英语：Currying），是把接受多个参数的函数变换成接受一个单一参数的函数，并且返回接受余下的参数而且返回结果的新函数的技术。

  • 原函数
  • 柯里化函数，调用 add 接收一个参数，返回 _add 函数，_add 函数接收一个参数并且通过闭包访问到 add 函数的参数。

装饰器

概括的讲，装饰器的作用就是为已经存在的函数或对象添加额外的功能。

其实装饰器就是一个函数，装饰器可以以不入侵其他函数为其他函数增加功能，它接收函数为参数并返回一个装饰过的函数（或其他对象）。

常用于有切面需求的场景，比如：插入日志、性能测试、事务处理、缓存、权限校验等场景。

装饰器是解决这类问题的绝佳设计，有了装饰器，我们就可以抽离出大量与函数功能本身无关的雷同代码并继续重用。

  • 原代码
  • 基础装饰器
  • 装饰器语法糖，@ 是上面的装饰过程的一种语法糖，简化了装饰器的使用

元信息问题

  • 装饰器装饰了原程序，但是程序的元信息会被覆盖，所以当我们查看函数名的时候，会返回装饰后的那个函数的名称
  • 内置模块 functool 有个功能 swraps，swraps 它也是一个装饰器，它能把原函数的元信息拷贝到装饰器函数中，使装饰器函数与原函数有一样的元信息了。
