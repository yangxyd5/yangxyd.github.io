Tornado 笔记

发表于 2019-04-28 | 分类于 Python

最近想实现一个 webshell 需要用到 WebSocket 技术，所以学学 Tornado

Hello World

tornado.httpserver

  • 一个非阻塞的单线程HTTP服务器，它可以启动多个进程。
  • debug = True 参数有以下作用

tornado.options

tornado 的命令行参数解析器，它的 define 方法负责定义接收的参数，options 则是接收并且解析完成后的参数，可以使用 options.parse_command_line 解析命令行参数，或者使用 options.parse_config_file 解析配置文件中的参数

  • 从命令行读取配置
  • 从配置文件读取变量
  • 直接使用 python 模块来当作配置

tornado 模块化

  • 单应用项目文件组织
  • index.py 视图文件
  • config.py 配置文件
  • app.py 应用对象，视图和路由在这里配置
  • server.py 服务器程序文件

tornado 路由

路由 模式
(r”/“, index.MainHandler) 绝对路径匹配
(r”/“, index.MainHandler,{“key”: “value”}) 向视图传递参数，在视图函数的 initialize 方法中可以接收 key 这个参数，然后赋值 self.key = key
(r”/\d+”, index.MainHandler) 正则表达式匹配
tornado.web.url(r”/url”, index.Url, {“key1”: “value1”, “key2”: “value2”}, name=”urlfor”) 定义命名的路由条目，可以在视图中使用 self.reverse_url(name=urlfor)，获得匹配正则为 urlfor 的 url 地址
(r”/index/(?P\<page>\d*)”, index.MainHandler) 正则表达式分组匹配
(r”/request/(\w+)/(\d+)”, request.MainHandler),

RequestHandler 钩子

继承 RequestHandler 并重写下面的方法，可以在 HTTP 请求到达时，分别用不同的方法来处理 HTTP 的请求

initialize

重写这个方法，可以接收路由向视图传递的参数，

  • 首先在定义路由时候传递参数给视图
  • 在视图类的 initialize 方法中接收并赋值

prepare

prepare 钩子方法是一个预处理方法，在执行对应的请求之前调用

set_default_headers

set_default_headers 它在进入 HTTP 响应处理方法之前被调用，可以重写该方法预先设置默认的 headers。

如果在 get、post 方法内又调用了 self.set_header 那么 get 、post 方法内设置的会覆盖 set_default_headers 设置的头部信息

  • 例如下面，无论通过那个方法响应客户端，都会带有 set_default_headers 设置的头部

write_error

它接收 self.send_error 抛出的错误状态码，和一些信息，处理并返回页面

on_finish

在请求结束之后进行调用。

RequestHandler 方法

继承 RequestHandler 后可以直接使用一些已经存在的方法

self.set_header

  • 向响应报文设置一个头部

self.set_status

  • 设置响应报文的响应码和响应短语

self.redirect

  • 将请求重新向到新页面，permanent 表示是否永久重定向

self.send_error

  • 抛出一个错误状态码，tornado 会调用 write_error 方法进行处理并响应这个错误

self.reverse_url

  • 命名路由使用 tornado.web.url 来定义
  • 在视图中使用 self.reverse_url 反向解析一个已经命名的路由条目

self.get_query_argument

self.get_query_argument，可以取得通过 get 方法请求的查询参数，name 指明需要接收的参数名，default 指明没有接收到这个参数时候的默认值，strip 指明是否过滤参数前后的空格。

  • self.get_query_arguments 接收来自 get 请求传递的参数，如果存在同名参数，则取得全部。
  • self.get_query_argument 接收来自 get 请求传递的参数，如果存在同名参数，则取最后一个。

self.get_argument

self.get_argument 可以取得通过 get、post 方法请求提交的数据，name 指明需要接收的参数名，default 指明没有接收到这个参数时候的默认值，strip 指明是否过滤参数前后的空格。

  • self.get_argument 接收来自 get、post 请求传递的参数，如果存在同名参数，则取最后一个。
  • self.get_arguments 接收来自 get、post 请求传递的参数，如果存在同名参数，则取得全部。

self.request

self.request 包含本次请求的所有信息

  • 通过 POST 提交参数和文件，查看 self.request 信息

self.request.files

self.request.files

  • 下面命令提交了 2 个表单，分别是 key 和 files，key 有两个值，files 有两个文件
  • self.request.files 对象中保存了本次请求所提交的文件，下面将上传的文件保存在本地

self.set_cookie()

  • 设置一个明文 Cookie 并设定过期时间为 3 小时后

self.set_secure_cookie()

  • 设置一个加密的 Cookie 并设定过期时间为 3 小时后

self.clear_cookie

  • self.clear_cookie 清除指定 Cookie，self.clear_all_cookies 清除所有的 Cookie

基于协程的异步

tornado.httpclient.AsyncHTTPClient

tornado 提供的异步 web 请求客户端，可以用来进行异步 web 请求

  • fetch

fetch(request) 用于执行一个 web 请求，并异步返回一个 tornado.httpclient.HTTPResponse，request 可以时一个 url，也可以是一个 tornado.httpclient.HTTPRequest 对象。

  • HTTPRequest
  • HTTPResponse

从 Tornado 3.1 版本开始，无需 @asynchronous，即仅需要使用 @gen.coroutine 就可以了，而且 @gen.coroutine 装饰的代码段执行完成后会自动调用finish。

gen 模块的某些较旧部分已弃用，将在 6.0 中删除。 这包括 engine, YieldPoint, Callback, Wait, WaitAll, MultiYieldPoint, Task。

@asynchronous 装饰器已经过时，在 6.0 中删除。

  • 协程实现的异步处理请求

WebSocketHandler

  • self.write_message(message, binary=False)

主动向客户端发送 message 消息，message 可以是字符串或者字典（自动转为 JSON 字符串），如果 binary 为 False ，则 message 会以 UTF-8 编码发送，如果 binary 为 False 则发送二进制模式，字节码。

  • self.close

关闭 websocket 连接

  • self.open

当 websocket 连接被建立时调用

  • self.on_message

当客户端发来消息时调用

  • self.on_close

当 websocket 连接关闭后调用

  • self.check_origin

判断源客户端，对于符合条件的请求允许连接，相当于是同源策略，返回布尔值决定是否允许连接

Websocket 笔记

发表于 2019-04-27 | 分类于 web

socket.io

由于 Flask-SocketIO 这个库它只支持 socket.io 协议的客户端，不支持原生的 websocket 所以需要学习以下 socket.io 的使用。

socket.id

websocket 套接字的唯一标志，在 connect 事件触发后才有值。

socket.connected

websocket 套接字是否已经建立连接，布尔值

socket.disconnected

websocket 套接字连接是否已经断开，布尔值

socket.open

手动打开套接字，socket.connect 是它的同义词

或者监听断开事件进行重连

socket.emit

将数据发送到由字符串名称标识的套接字，数据可以是任何可以序列化的数据结构。
